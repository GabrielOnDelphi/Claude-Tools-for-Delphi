Brief history
Recent months
Gabriel has been deeply engaged with Delphi 13/FMX development across several technical areas: debugging Win32-to-Win64 migration issues (particularly around uninitialized variables, pointer casting, and enum size behavior), resolving FireMonkey runtime style switching bugs. He also investigated platform service retrieval patterns in FMX, favoring SupportsPlatformService for its defensive error handling. He set up Claude Code in a WSL environment. He has a clear preference for FreeAndNil over Free or DisposeOf, avoids deprecated Delphi constructs, and favors modern, safe coding patterns.

Earlier context
Gabriel created a custom "delphi-modern" skill to enforce consistent, hallucination-free Delphi code generation — codifying rules around memory management, forbidden constructs (absolute keywords, raw pointers, Application.ProcessMessages), minimal generics usage, and FMX/VCL patterns. He also worked through WSL installation issues on Windows 10 (NTFS compression errors, npm permission setup) as part of getting Claude Code running.
